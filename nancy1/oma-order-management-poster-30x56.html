OMA

React Native Order & Sales Management App

Mobile-first workflow for orders, approvals, dispatch, customer ledgers, and live business analytics.

60 secTarget order creation
< 2 minApproval handoff
1 appSales to warehouse loop

Abstract

OMA is a React Native mobile application that modernizes the order lifecycle for distributors and wholesalers. It replaces notebook-to-desktop workflows with mobile order creation, manager approvals, dispatch tracking, customer ledger visibility, and real-time analytics.

The project shows how one shared mobile workspace can connect sales representatives, managers, and warehouse teams in a faster operating loop.

Objectives

  • Digitize and automate field order creation.
  • Provide embedded customer ledger and credit visibility.
  • Support approval, recheck, dispatch, and fulfillment workflows.
  • Enable real-time order status tracking across roles.
  • Improve coordination between sales, managers, and warehouse users.

Problem / Introduction

  • Sales representatives often record orders manually and re-enter them later.
  • Managers need customer risk, ledger exposure, and terms before approval.
  • Warehouse teams rely on delayed updates for dispatch readiness.
  • Existing workflows lack real-time visibility into approvals and receivables.

OMA solves these bottlenecks through a connected mobile workflow for create, review, fulfill, and analyze operations.

Technology Stack

FrontendReact Native, Expo, TypeScript, Expo Router
BackendNode.js, Express.js, REST APIs
Cloud DataGoogle Sheets API and workbook tabs
SecurityRole-based demo login and JWT-ready APIs

Workflow / Process Flow Chart

Create OrderCapture customer, products, quantities, and notes.
SubmitSend the order for manager review.
Manager ReviewCheck value, credit, ledger, and terms.
Approve / RecheckApprove fulfillment or request changes.
WarehousePick, pack, and prepare approved items.
DispatchShip order and update fulfillment status.
AnalyticsMonitor revenue, receivables, alerts, and status.

Results / Outcome

  • Order creation target under 60 seconds.
  • Approval workflow target under 2 minutes.
  • Less duplicate entry between sales and dispatch.
  • Clear visibility for managers and workers.

Conclusion

OMA demonstrates a practical mobile-first approach for order and sales management. It improves speed, transparency, and coordination by connecting creation, approval, dispatch, client records, and analytics in one workflow.

Future Scope

  • Offline order capture and synchronization.
  • Push notifications for approvals and dispatch.
  • ERP / Tally / CSV integration pipeline.
